About ETF

ETF is an open source testing framework for validating spatial data, metadata and web services in Spatial Data Infrastructures (SDIs). It can be used via a responsive web application or via a REST API.

The design of ETF is driven by three goals: be user-friendly, consistent with the standards and capable of testing all resources in an SDI.

interactive instruments has developed ETF and is the primary maintainer of the software.

Prerequisites

  • OpenJDK 11 is installed

  • Git installed

  • Apache Tomcat 9 installed

  • IntelliJ installed

  • Smart Tomcat plugin installed in IntelliJ

Cloning the source code

Clone the Source code from the GitHub repository.

git clone -b next https://github.com/etf-validator/etf-webapp.git
The next branch contains the latest developments.

Using IntelliJ to run the etf-webapp

Creating the project

Open Intellij.

Open the "File" tab.

Select "New".

Open "Project from existing sources…​".

Select the etf-webapp project.

Select the following options in the new window that appeared: - Import project from external model - Gradle

Press continue.

At this moment, the README should have been displayed in the window.

Creating a run configuration file (etf-webapp)

Open the dropdown menu, in which appear by default "Current File".

Select "Edit Configurations…​".

A new window will appear.

Click add new.

Select Smart tomcat.

Fill it in like in the next example.

Vm options: -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005

Modifying the JDK used

Open the "File" tab.

Select "Project structure"

SDK option will appear as no SDK

Select the SDK version 11

If you want to test it out, you can run it with this configuration, and it should work in localhost:8080/etf-webapp

Debugging using IntelliJ

Creating a project for the testdrivers

Clone the testdrivers from the following links:

  • etf-bsxtd ETF test driver for running BaseX XQuery based test projects

  • etf-tetd ETF test driver for executing Test Suites on a remote OGC TEAM Engine instance

  • etf-suitd ETF test driver for running SoapUI based test projects

Open IntelliJ.

Open the "File" tab.

Select "New"

Open project from existing sources

Select the testdriver project (Select only one of them)

Creating a run configuration file

Open the dropdown menu, in which appear by default "Current File".

Select "Edit Configurations…​".

A new window will appear.

Click in "Add new".

Select "Remote JVM Debug".

Fill it in like in the following example:

In Command line arguments for remote JVM, you have to put the same as in vm options of the webapp

Modifying the JDK used

Open the "File" tab.

Select "Project structure"

SDK option will appear as no SDK

Select the SDK version 11

You can use 1.8 or 11, it depends on the modules you want to debug. In IntelliJ, it is also possible to use different SDKs in different modules, in case you need it.

Adding the ETS repository and testdrivers

Download the ETS repository from the release

Extract it.

Run the etf-webapp once.

It will create a folder called ".etf".

Copy the extracted files in the "projects" folder, inside ".etf".

Put the testdrivers in td/lib/

Run again the etf-webapp.

You’ll find all the ETS inside now.

How to use

Open both projects in different windows

Run the etf-webapp

Debug in the testdrivers project.

If you want to debug another test driver or anything contained, you just need to add a new module with the contents of the project you want to debug in the project with the testdrivers.
